我的第一次提交项目到GitHub

Posted by 张学峰 on 2017-03-27

你想让你的博客长草还是长粉？ —-自己问自己

想一次性成功？没门！

今天想着把之前写的一些小东西放到GitHub上去吧，于是就开始折腾上传项目。Google上的教程还是很多的

但是坑也是很多的。今天先把自己遇到的坑说一下吧。

- **不是**每个项目要有自己的SSH！一个SSH就是可以通用。 当然你如果是一个项目分别在GitHub上  <br>也在公司另外还存放在别的地方，那么你可以申请多个SSH。申请的时候会提示你覆盖原有的，你可以<br>
在.ssh下新建一个文件夹并自行命名，然后分别存储多个SSH。<br>
- SSH出现问题<br>SSH:connect to host github.com port 22:connection timed out.<br>
  解决方法：
  ![](port22.png)<br>
- fatal:Not a git repo (or any of the parent directories): .git [duplicate]<br>
  ![](fatal.png)<br><br>
- git add .的时候git bash 出现提示：LF will be replaced by CRLF in git .<br>
- 以上解决方案均在Stack Overflow上获取，如有雷同，那就握个手呗. <br>

嗯哼，我还是顺利提交了

我的步骤是：

- 在GitHub上新建一个仓库 yourRepo。
- git clone git@github.com:Yourname/yourRepo.git  <br>
  或者像我一样更改过URL的就这样写：git clone https://github.com/Yourname/yourRepo.git
- git add .
- git commit -m "自己爱写啥写啥，描述自己的项目"
- git pull origin master (如果你是和我一样是新建仓库 然后直接上传代码，这一步其实没啥用)
- git push origin master (结束！)
第一次记录自己的折腾过程到博客，感觉还不错。以后继续折腾~~
